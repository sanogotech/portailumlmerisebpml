Pourquoi apprendre UML, Merise et BPMN ?

Ces langages de modélisation sont essentiels pour analyser, concevoir et documenter les systèmes d'information de manière rigoureuse et standardisée.

UML (Unified Modeling Language)

Introduction à UML

UML est un langage de modélisation standardisé pour la conception orientée objet. Il offre 14 types de diagrammes répartis en 2 catégories :

Diagrammes structurels :

  • Diagramme de classes
  • Diagramme d'objets
  • Diagramme de composants
  • Diagramme de déploiement
  • Diagramme de paquetages
  • Diagramme de structure composite
Diagramme de Classes

Le diagramme de classes est le plus utilisé en UML. Il représente :

  • Les classes avec leurs attributs et méthodes
  • Les relations entre classes (héritage, association, etc.)
  • Les interfaces et les énumérations

Diagramme de Séquence

Montre les interactions entre objets dans le temps sous forme de messages.

Utile pour modéliser les cas d'utilisation et le comportement du système.

Diagramme d'Activité

Représente les flux de travail et les processus métier.

Similaire aux organigrammes mais avec une sémantique plus riche.

Diagramme de Cas d'Utilisation

Décrit les fonctionnalités du système du point de vue des utilisateurs.

Identifie les acteurs et leurs interactions avec le système.

Méthode Merise

Concepts de base

Merise est une méthode de conception de systèmes d'information basée sur trois niveaux :

Niveaux d'abstraction :

  1. Conceptuel : Modèle indépendant de toute technique
  2. Logique : Adaptation au type de système
  3. Physique : Implémentation concrète
Modèle Conceptuel de Données (MCD)

Le MCD représente les données et leurs relations indépendamment de toute implémentation.

CLIENT
COMMANDE
1,n
Modèle Logique de Données (MLD)

Transformation du MCD en modèle adapté au type de base de données (relationnel, réseau, etc.).

Normalisation des tables et définition des clés.

Modèle Physique de Données (MPD)

Implémentation concrète dans un SGBD spécifique.

Définition des types de données, index, procédures stockées, etc.

Modèle Organisationnel des Traitements (MOT)

Décrit les processus métier et leur organisation.

Spécifie qui fait quoi, quand et comment.

BPMN (Business Process Model and Notation)

Introduction à BPMN

BPMN est le standard pour la modélisation des processus métiers. Il permet de :

  • Documenter les processus existants
  • Concevoir de nouveaux processus
  • Faciliter la communication entre métiers et techniques
  • Préparer l'automatisation des processus

Création d'un MCD Merise

Cette simulation vous guide dans la création d'un modèle conceptuel de données pour un système de gestion de commandes.

1. Identifier les entités

Les entités principales identifiées :

  • CLIENT
  • COMMANDE
  • PRODUIT
  • LIVRAISON
2. Définir les associations

Associations identifiées :

  • Un CLIENT passe une ou plusieurs COMMANDES (1,n)
  • Une COMMANDE concerne un ou plusieurs PRODUITS (1,n)
  • Une COMMANDE génère une LIVRAISON (1,1)
3. Ajouter les attributs

Attributs principaux :

EntitéAttributsIdentifiant
CLIENT id_client, nom, prénom, adresse id_client
COMMANDE num_commande, date, montant num_commande
4. Vérifier les règles de gestion

Règles de gestion validées :

  • Un client peut avoir plusieurs adresses ? → Nouvelle entité ADRESSE
  • Un produit peut être commandé sans stock ? → Ajout attribut "disponible"
  • Une commande peut être annulée ? → Ajout attribut "statut"
